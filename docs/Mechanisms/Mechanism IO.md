
# Mechanism IO

The `MechanismIO` interface is used to declare the inputs and outputs that a mechanism will use. First, the autogenerated file will contain a `@Fallback`, this is a feature from MARS that avoids null-pointers during the code execution. Then, a class of `MechanismInputs` is created to declare the variables that you will use, while the functions your mechanism will have will be defined below it. 

Here is an example of a `MechanismIO` interface for an arm mechanism:

```java
@Fallback
public interface ArmIO extends IO<ArmIO.ArmInputs>{

    public static class ArmInputs extends Data<ArmInputs>{

        public double position = 0;
        public Rotation2d rotation = new Rotation2d();
        public double targetAngle = 0;

    }

    public void applyOutput(double volts);

    public void setPosition(double angle);

}    
```
